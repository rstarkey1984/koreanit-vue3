# Node.js 실행 환경과 Express API 서버

이 문서는 프로젝트에서 사용할 **Node.js 실행 환경**을 준비하고,
**Express 기반 API 서버**를 직접 만들어 JSON 응답까지 확인하는 것을 목표로 한다.

---

## 1. Node.js는 무엇인가

Node.js는 **브라우저 밖에서 JavaScript를 실행하는 런타임(실행 환경)** 이다.
HTML/DOM을 다루는 용도가 아니라, **서버 프로그램과 API**를 만들기 위해 사용한다.

Node.js가 제공하는 대표 기능

* HTTP 요청/응답 처리
* 외부 서버로의 네트워크 요청
* 파일 시스템 접근
* 서버 프로그램의 지속 실행

이 프로젝트에서 Node.js는
**프론트엔드(Vue)가 사용할 데이터를 제공하는 API 서버** 역할을 담당한다.

---

## 2. nvm을 사용하는 이유

Node.js는 버전 변화가 잦고, 프로젝트마다 요구 버전이 다를 수 있다.

nvm(Node Version Manager)은 다음 문제를 해결한다.

* Node.js 여러 버전 설치 가능
* 프로젝트별 Node 버전 전환 가능
* 시스템 환경 오염 방지

---

## 3. nvm 설치 및 Node.js 설치

### 3-1. nvm 설치

터미널에서 다음 명령을 실행한다:
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
```

`.bashrc` 환경변수 적용:
```
source ~/.bashrc
```

정상 설치 확인:
```bash
nvm -v
```

버전 번호가 출력되면 설치 완료
```
0.39.7
```
### 3-2. Node.js(LTS) 설치

```bash
nvm install --lts
```

설치 확인:
```bash
node -v
npm -v
```

체크 포인트

* Node와 npm이 함께 설치된다
* Node.js는 “프로그램”이 아니라 **실행 환경**이다
* npm은 Node Package Manager의 약자로, 전 세계 개발자들이 미리 만들어 놓은 패키지를 내 프로젝트에 간편하게 가져와 쓸 수 있게 해주는 도구입니다.

## 4. Node.js 실행 방식 체험

JavaScript를 **브라우저가 아닌 터미널에서 실행**해본다.

### 4-1. 터미널에서 바로 실행(REPL)

```bash
node
```

Node 콘솔이 열리면 다음을 입력한다.

```js
console.log("hello node");
```

종료

* `Ctrl + C` 두 번

핵심 인식

* 문법은 같지만 실행 환경이 다르다
* DOM/화면 없이 **로직만 실행**된다

### 4-2. .js 파일 만들어 실행하기

작업 폴더 생성

```bash
mkdir -p ~/projects/koreanit/node-basic
cd ~/projects/koreanit/node-basic
```

파일 생성

```bash
touch index.js
```

`index.js` 내용

```js
console.log("Node.js 파일 실행 테스트");

const a = 10;
const b = 20;

console.log("a + b =", a + b);
```

실행

```bash
node index.js
```

체크 포인트

* HTML/DOM 없이 파일 단위 프로그램으로 실행된다

## 5. Express로 API 서버 만들기

이제 Node.js를 이용해 **요청을 기다리는 서버 프로그램(API 서버)** 을 만든다.

### 5-1. 프로젝트 폴더 준비

```bash
mkdir -p ~/projects/koreanit/news-api
cd ~/projects/koreanit/news-api
```

### 5-2. npm 프로젝트 초기화

```bash
npm init -y
```

`package.json`이 생성된다.

### 5-3. Express 설치

```bash
npm install express
```

---

## 6. 서버 코드 작성

`server.js` 생성

```bash
touch server.js
```

`server.js` 내용

```js
const express = require("express");

const app = express();
const PORT = 3000;

// 서버 상태 확인용
app.get("/", (req, res) => {
  res.send("API 서버 실행 중");
});

// 뉴스 API (지금은 샘플 JSON 응답)
app.get("/api/news", (req, res) => {
  const data = {
    source: "sample",
    count: 3,
    items: [
      {
        id: 1,
        title: "샘플 뉴스 1",
        link: "https://example.com/1",
        publishedAt: "2026-01-12 10:00:00",
      },
      {
        id: 2,
        title: "샘플 뉴스 2",
        link: "https://example.com/2",
        publishedAt: "2026-01-12 11:00:00",
      },
      {
        id: 3,
        title: "샘플 뉴스 3",
        link: "https://example.com/3",
        publishedAt: "2026-01-12 12:00:00",
      },
    ],
  };

  res.json(data);
});

app.listen(PORT, () => {
  console.log(`API 서버 실행: http://localhost:${PORT}`);
});
```

---

## 7. 서버 실행 및 확인

실행

```bash
node server.js
```

브라우저에서 확인

* `http://localhost:3000/` → “API 서버 실행 중”
* `http://localhost:3000/api/news` → JSON 응답

체크 포인트

* HTML 화면이 아니라 **JSON 데이터만** 응답한다
* `/api/news` 같은 URL(엔드포인트)을 만들고 요청이 오면 데이터를 준다
